## Новые чеки (4 штуки, 42 → 46)

### 0l. AI API cost protection [core]
- **Слой**: 0 (Безопасность) → SECURITY-EXTRA.md
- **Проблема**: Вайбкодеры используют OpenAI/Anthropic API без лимитов. Зацикленный агент = $500 за ночь.
- **Проверяет**: billing alerts настроены, max_tokens указан в вызовах, dev/prod ключи раздельные
- **Детекция**: `OPENAI_API_KEY`/`ANTHROPIC_API_KEY` в .env.example, grep `max_tokens` в коде, проверка использования одного ключа

### 0m. Backup strategy [advanced]
- **Слой**: 0 (Безопасность) → SECURITY-EXTRA.md
- **Проблема**: Git remote есть (0a), но данные БД и файлы хранилища не бэкапятся.
- **Проверяет**: managed DB (Railway/Supabase/RDS → бэкапы включены), или backup скрипт существует
- **Детекция**: `pg_dump`/`mongodump` скрипты, Railway/Supabase в конфигах, cron задачи

### 1g. Database migration tool [quality]
- **Слой**: 1 (Фундамент) → FOUNDATION-EXTRA.md (новый файл)
- **Проблема**: Вайбкодеры меняют схему вручную ALTER TABLE. Потеря данных, невоспроизводимость.
- **Проверяет**: alembic/prisma migrate/knex migrations/typeorm настроен
- **Условие**: Только если проект имеет БД (DATABASE_URL, asyncpg, prisma, etc.)
- **Детекция**: `alembic/`, `prisma/migrations/`, `migrations/`, alembic.ini, prisma/schema.prisma

### 2l. Error monitoring [core]
- **Слой**: 2 (Качество) → QUALITY-EXTRA-2.md (новый файл)
- **Проблема**: Вайбкодеры деплоят и не видят ошибки. Узнают от юзеров.
- **Проверяет**: Sentry/LogRocket/Axiom/Datadog настроен, SENTRY_DSN в env
- **Детекция**: `@sentry/node`, `sentry-sdk`, `SENTRY_DSN`/`LOGROCKET_APP_ID` в .env.example, sentry.*.config.*
